## Ejecutar un Flujo

Una vez creado y configurado tu flujo, necesitas probarlo para asegurarte de que funcione correctamente antes de ponerlo en producción.

## Opciones de Ejecución

### 1. Modo Test
- Permite ejecutar el flujo manualmente en modo prueba.
- No requiere activarlo permanentemente.
- Ideal para depuración o ajustes finos.

#### ¿Cómo probar?
1. En el editor de flujo, haz clic en **Test**.
2. Proporciona los datos de prueba si es necesario (por ejemplo, JSON simulado).
3. Observa la ejecución paso a paso.
4. Revisa el resultado en el historial de ejecución temporal.

### 2. Activar el Flujo
- Deja el flujo disponible para ser activado automáticamente por Webhooks, Schedules o Triggers.
- Permite ejecuciones reales.

#### ¿Cómo activar?
1. Guarda todos los cambios del flujo.
2. Haz clic en **Activar** en la cabecera del editor o desde la lista de flujos.
3. El flujo queda **online** y listo para operar.

---

## Precauciones antes de activar
- Asegúrate de que todos los nodos estén configurados.
- Revisa credenciales necesarias para conexiones externas.
- Valida que los caminos de error estén correctamente definidos (ej: nodos Decision).

---

## 📜 ¿Dónde ver las ejecuciones?

- Accede a la sección **Historial** para consultar:
  - Flujos exitosos
  - Flujos con error
  - Logs detallados de cada paso
